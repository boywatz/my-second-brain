# Claude Project Instructions — Second Brain (Obsidian)

## Context
I am building a Second Brain using Obsidian, stored in a local folder connected to a Git remote for backup.

## What Has Been Set Up
- **PARA folder structure:** `00_Inbox`, `01_Projects`, `02_Areas`, `03_Resources`, `04_Archive`, `05_Assets`
- **Templates folder** with 6 templates: Daily Note, Fleeting Note, Meeting Note, Book Note, Project Note, MOC Template
- **Key files:** `Home.md` (dashboard), `Getting Started.md` (workflow guide), `Recommended Plugins.md`
- **Obsidian settings configured:** new files go to `00_Inbox`, attachments go to `05_Assets/Attachments`, Daily Notes use the Daily Note template

## Design Principles
- **PARA Method** — organize by actionability, not topic (Projects → Areas → Resources → Archive)
- **Zettelkasten** — atomic notes, one idea per note, link with `[[]]` instead of deep folder nesting
- **MOC (Map of Content)** — create index notes when a topic accumulates many notes
- **Organic Growth** — capture first in Inbox, organize later; use tags for status, not topic

## Note Status Tags
- `#to-process` — captured but not yet organized
- `#permanent` — finalized, evergreen knowledge
- `#fleeting` — quick idea, pending review
- `#moc` — Map of Content index note
- `#active` — currently in use
- `#review` — needs revisiting

## Language
Mixed Thai and English — structure/labels in English, content freely in Thai or English.

## Preferred Workflow
When helping with this Second Brain:
1. Suggest where new notes belong in the PARA structure
2. Recommend linking to existing notes with `[[note name]]`
3. Suggest relevant tags from the list above
4. If a topic has many notes, suggest creating a MOC
5. Encourage capturing quickly in Inbox first, organizing later

## Recommended Plugins (to install)
Priority order: Templater → Calendar → Obsidian Git → Dataview → Quick Add

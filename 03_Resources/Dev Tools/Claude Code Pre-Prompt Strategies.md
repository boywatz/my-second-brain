# Claude Code Pre-Prompt Strategies

#permanent #dev-tools #claude-code #ai-workflow

---

## Overview

Pre-Prompt คือการวาง System Prompt หรือ Instructions ไว้ก่อนเริ่ม Session ของ Claude Code เพื่อ "ตั้งค่าบุคลิก" และกำหนดรูปแบบการทำงานของ AI ล่วงหน้า ทำให้ AI เขียนโค้ดและตัดสินใจได้ตรงกับสิ่งที่เราต้องการมากขึ้น

มีสองแนวทางหลักที่นิยมใช้กันในชุมชน:

---

## 1. Get Shit Done (GSD)

**จุดเด่น:** ประสิทธิภาพและความเร็ว

### ปรัชญา
GSD ออกแบบมาเพื่อให้ Claude ทำงานให้เสร็จเร็วที่สุด ลดขั้นตอนที่ไม่จำเป็น และแก้ปัญหา **Context Rot** — ปรากฏการณ์ที่ AI เริ่ม "หลงลืม" หรือด้อยคุณภาพลงเมื่อ Context Window เริ่มเต็ม

### เหมาะสำหรับ
- งานด่วน แก้ปัญหาเฉพาะจุด
- งาน Task-oriented ที่ต้องการผลลัพธ์ทันที
- เคลียร์ backlog หรือแก้บั๊กที่ค้างอยู่
- Session สั้นที่ไม่ต้องการ context ยาว

### ข้อควรระวัง
- อาจขาดความละเอียดในเชิงโครงสร้างระยะยาว
- ไม่เหมาะสำหรับการวางสถาปัตยกรรมระบบใหม่

---

## 2. Everything Claude Code

**จุดเด่น:** ความครอบคลุมและการวางระบบ

### ปรัชญา
Everything Claude Code เป็นโซลูชันที่รวม การวางแผน → การออกแบบสถาปัตยกรรม → การเขียนโค้ด ไว้ในแนวทางเดียวกัน เน้นการเขียน Spec และการออกแบบล่วงหน้าเพื่อไม่ให้ AI สับสนระหว่าง session

### เหมาะสำหรับ
- เริ่มโปรเจกต์ใหม่ตั้งแต่ต้น
- วางโครงสร้างโค้ดที่ซับซ้อน (Project-oriented)
- ทีมที่ต้องการรักษามาตรฐานของทั้งระบบ
- งานที่ต้องการ consistency ในระยะยาว

### ข้อควรระวัง
- ใช้ Token และเวลาประมวลผลสูงกว่า
- อาจ overkill สำหรับงานเล็กหรืองานด่วน

---

## ตารางเปรียบเทียบ

| คุณสมบัติ | Get Shit Done (GSD) | Everything Claude Code |
|---|---|---|
| **จุดเด่นหลัก** | ความเร็ว, ประสิทธิภาพ | ความครบถ้วน, ครอบคลุม |
| **การจัดการบริบท** | แก้ปัญหา Context Rot | วาง Spec ป้องกัน AI สับสน |
| **เหมาะกับ** | งานด่วน, แก้ปัญหาเฉพาะจุด | สร้างโปรเจกต์ใหม่, วางระบบ |
| **Token Usage** | ต่ำกว่า | สูงกว่า |
| **ความเสี่ยง** | โครงสร้างระยะยาวอาจไม่แน่น | อาจช้าและ overkill |

---

## แนวทางการเลือกใช้

```
เริ่มโปรเจกต์ใหม่?
  └─ YES → Everything Claude Code (วางรากฐานให้แน่น)
  └─ NO  → มี task ค้างที่ต้องเคลียร์?
              └─ YES → Get Shit Done (เร็ว, ตรงจุด)
              └─ NO  → พิจารณาตามขนาดและ scope ของงาน
```

**หมายเหตุสำคัญ:** ไม่ควรใช้ทั้งสองพร้อมกัน เพราะ Instructions ที่ขัดแย้งกันจะทำให้ AI สับสนและประสิทธิภาพลดลง (Prompt Conflict)
